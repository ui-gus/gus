Join Group

  • Actors: User
  • Goals: user will be able to request to join group
  • Preconditions: user is logged onto GUS
  • Summary: Related use cases:
  • Steps:
    1. User enters group name
    2. User clicks on the join button
    3. User identifies which subgroups they want to join
    4. User selects save and exit
  • Alternatives: User cancels out of use case by clicking on a different navigational area or clicking Cancel
  • Postconditions: user's request is auto approved, or request sent to group, user is added to group (if approved), and user is returned to the group's page
